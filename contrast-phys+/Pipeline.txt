# Contrast-Phys+ Pipeline

## 1. 30fps

    train.py: fs=30 固定   →   T=300 (10s)   →   bandpass/FFT 按 30Hz
         ↑                        ↑
    论文/UBFC 常用 30fps     推理也用 config.fs=30
    非 30fps 视频 → 警告，建议重采样或录 30fps

  参数: fs=30 采样率(Hz); T=300 时间帧数(10s); 带通 0.6–4 Hz ≈ 36–240 BPM。
  原因: 数据集多为 30fps；10s 便于时域设计；30Hz 下频率分辨率约 2 BPM。


FFT 是否影响 HR？	是，FFT 分辨率和选峰直接决定 HR 数值.
所以使用更长的时间进行visual其实就是提高了fft的分辨率, 让peak更加突出, 这就是为啥60s的peak比10s的peak更加突出而且准确

## 2. Training

    ┌─────────────────────────────────────────────────────────────────┐
    │  H5: imgs [N,128,128,3] RGB 0-255,  bvp [N]                      │
    └───────────────────────────┬─────────────────────────────────────┘
                                │ H5Dataset 随机截 T=300
                                ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │  transpose(3,0,1,2) → [C,T,H,W] float32   (不/255)               │
    └───────────────────────────┬─────────────────────────────────────┘
                                │ PhysNet(S=2,in_ch=3), ContrastLoss
                                ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │  fs=30, T=300, delta_t=150, K=4   →   results/<exp>/             │
    └─────────────────────────────────────────────────────────────────┘

  参数:
    fs=30, T=300, delta_t=150, K=4  |  时间维与 rPPG 采样
    S=2, in_ch=3                    |  PhysNet 空间块 2×2，RGB
    ContrastLoss 带通 40–250 BPM    |  约 0.67–4.17 Hz，与推理带通接近
    batch_size=2, lr=1e-5           |  train.py 内可改
  说明: H5 的 imgs 已为人脸裁剪 128×128；训练不做 /255，模型内 (x-mean)/std。

## 3. Inference（单次 HR）

    [T,128,128,3] 0-255
           │
           │  取最后 300 帧  (或 --full-waveform: win=300, stride=150 滑窗)
           ▼
    transpose + newaxis  →  [1,3,T,128,128] float32
           │
           ▼
    PhysNet  →  output[:,-1,:]  →  rPPG [T]
           │
           ▼
    butter_bandpass(0.6, 4, fs=30)  →  Hann  →  FFT 幅度谱
           │
           ▼
    0.6-4 Hz 峰值  →  抛物线插值/谐波去除  →  HR (BPM)

  参数: 0.6/4 Hz 即 36–240 BPM；单次 HR 用 300 帧(10s)；滑窗时重叠 5s。
  代码: utils_inference.dl_model；HR 用 utils_sig.hr_fft / hr_fft_parabolic、estimate_hr。
  注意: 用 FFT 幅度谱取峰，非 PSD。

## 4. 本地视频 → Ubuntu 推理

    ┌──────────────┐    上传     ┌──────────────┐
    │ 本地录 30fps │ ────────►  │ my_video.mp4 │
    │ ≥10s         │            └──────┬───────┘
    └──────────────┘                   │
                                       ▼
              ┌────────────────────────┴────────────────────────┐
              │ 路径A: OpenFace → landmarks → 裁剪(同训练)       │
              │ 路径B: --source 视频 → Haar/中心裁剪 128²       │
              └────────────────────────┬────────────────────────┘
                                       ▼
    [T,128,128,3] RGB 0-255  取 300 帧(或滑窗)
                                       │
                                       ▼
              dl_model(model, frames)  →  rPPG  →  bandpass+FFT  →  HR
                                       (fs=30 来自 config.json)

  路径A: load_frames_from_video_with_openface；bbox=1.5*(maxy-miny)，y 外扩 0.2，resize 128。
  路径B: capture_frames + Haar/中心裁剪，BGR→RGB，128×128；与训练不完全一致，适合快速试。
  入口: live_predict_webcam.py --source <视频路径>；权重与 fs 来自 results/.../config.json + best_model.pt。
